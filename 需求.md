项目名称（暂定）：寻迹紫禁城 / 故宫图寻

项目定位：
基于故宫官方数字全景资源（pano.dpm.org.cn）制作一个 GeoGuessr 风格网页游戏。用户观察故宫全景场景，在故宫地图上点击猜测所在位置，系统根据误差计分，并展示相关建筑知识点。

核心玩法：

1. 随机抽取一个故宫场景 scene
2. 用户浏览全景图（360 viewer）
3. 在故宫地图点击猜位置
4. 系统计算点击坐标与真实坐标距离
5. 输出分数
6. 展示建筑名称、知识点、正确位置

已确认数据源：

1. 官方站点：
   https://pano.dpm.org.cn/

2. tile 资源示例：
   https://pano.dpm.org.cn/panoramas/20/krpano/panos/2803_summer.tiles/u/l3/06/l3_u_06_01.jpg

官方网站实现说明：

* 使用 krpano
* cubemap 六面：f/b/l/r/u/d
* 多级分辨率：l1/l2/l3
* tile 按需加载

3. tour.xml 已找到，包含 scene 配置：
   <scene name="scene_4_summer" title="太和殿左侧小门"...>

4. coordinate.json 已找到，包含坐标数据，例如：

{
"panorama_name": "太和门及前三殿区",
"scene_group_name": "太和殿平台",
"scene_id": 1,
"scene_name": "scene_1_summer",
"scene_title": "太和殿正殿",
"coordinate": "16.45424538694282,0.703125"
}

说明：
scene_title 可直接作为题目名称。
coordinate 可作为地图真实点位，需确认坐标系（经纬度 vs 笛卡尔）和单位（度 vs 米）。
